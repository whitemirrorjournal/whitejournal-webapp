# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

This repository (`whitejournal-webapp`) is a **static deployment repo** — it contains only the pre-compiled Flutter web output served by Vercel at `app.whitemirrorjournal.com`. There is no Flutter/Dart source code here.

The Flutter source code lives in a separate repository: `whitemirror-app`.

## Deployment Workflow

1. Build in the source repo:
   ```bash
   flutter build web --release
   ```
2. Copy `build/web/` contents into this repo.
3. Push to `main` — Vercel auto-deploys on push (`buildCommand: "exit 0"` means no build step; Vercel just serves the static files directly).

## Key Files

- **[vercel.json](vercel.json)** — Vercel configuration: SPA rewrites (all routes → `/index.html`), security headers (CSP, HSTS, X-Frame-Options), no framework build step.
- **[CNAME](CNAME)** — Custom domain binding for Vercel.
- **[index.html](index.html)** — Flutter web entry point generated by `flutter build web`.
- **[flutter_service_worker.js](flutter_service_worker.js)** — Service worker for offline caching; regenerated on each build.

## Security Headers (vercel.json)

The CSP allows connections to: Supabase, Vercel, Google Fonts, ElevenLabs API. Any new external API added in the source app must also be whitelisted here under `connect-src`.

## What NOT to edit manually

All files except `vercel.json` and `CNAME` are build artifacts — do not edit them by hand. Changes will be overwritten on the next `flutter build web --release` + copy cycle.
